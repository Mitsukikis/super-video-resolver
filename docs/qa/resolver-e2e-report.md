## API 用例结果

| 用例 | 结果 | 说明 |
| --- | --- | --- |
| 空链接 | 通过 | 返回“请输入视频链接” |
| 非 HTTP/HTTPS 链接 | 通过 | 返回格式错误提示 |
| 未支持平台链接 | 通过 | 返回“暂不支持该平台” |
| 未站内授权时提交平台 Cookie | 通过 | 返回“使用临时 Cookie 解析需要先登录” |
| YouTube 公开链接 | 通过 | 返回 `11` 条轨道、`7` 个下载选项 |
| YouTube 已不可用视频 | 通过 | 返回“Video unavailable”，未泄露服务器路径 |
| 站内授权后提交无效平台 Cookie | 部分通过 | 公开视频仍可解析，返回 1 条 Cookie 使用警告；未在报告中记录 Cookie |
| Bilibili 公开链接 | 未通过 | 源站返回 HTTP 412，当前需要后续适配或请求策略处理 |
| X / Twitter 公开帖 | 未通过 | 当前测试帖无可下载视频或需要账号态 Cookie |
| 网络超时 | 通过 | 使用 `RESOLVE_TIMEOUT_MS=1` 验证，返回“解析超时” |

## 安全检查

- Cookie 只通过本次请求提交；本报告未记录 Cookie 内容。
- 健康接口不返回 `YTDLP_PATH`、`FFMPEG_PATH` 或本机绝对路径。
- 解析失败时已过滤 Python dependency warning 和本机路径。
- 媒体直链只在用户解析结果页面中用于下载，不写入本报告。
- 浏览器合并不可用时不伪装为已合并完成。

## 当前阻塞

1. Bilibili 当前公开测试链接返回 HTTP 412，需要后续研究 yt-dlp 请求头、Cookie 或源站风控策略。
2. X / Twitter 需要更稳定的公开视频样本，部分帖子没有视频或需要账号态。
3. 浏览器本地合并受源站 CORS 限制，当前不能承诺所有平台都能在浏览器内完成合并。

## 验收结论

V2 工作台的 YouTube 公开解析、错误处理、依赖检查、超时处理和真实资源清单展示已通过本轮 E2E。Bilibili、X / Twitter 和浏览器本地合并仍存在真实阻塞，不能在产品文案中标记为完整可用。
