# 多平台解析工作台设计规格

## 目标

把当前单页解析器升级为类似高级 SaaS 仪表盘的多平台解析工作台。页面左侧提供平台模块导航，右侧根据所选平台显示对应的解析控制台、能力说明和状态提示。本轮只做前端信息架构和交互升级，不新增后端真实解析平台。

## 范围

本轮实现：

- 左侧固定平台导航。
- 平台切换后右侧内容实时更新。
- 已支持平台继续使用现有 `/api/resolve`。
- 未支持平台显示“即将支持/模块预留”，不提交解析请求。
- 保留现有粒子背景、深色高级视觉和访问码/Cookie 逻辑。

本轮不实现：

- 不接入抖音、小红书、微博、TikTok、Instagram、快手的真实解析。
- 不新增服务端下载、代理、缓存或合并。
- 不保存平台 Cookie。

## 平台列表

左侧平台导航包含：

- 全部平台：展示总体状态和通用解析入口。
- X / Twitter：已支持，使用现有 `x` resolver。
- Bilibili：已支持，使用现有 `bilibili` resolver。
- YouTube：已支持，使用现有 `youtube` resolver。
- 抖音：即将支持，模块预留。
- 小红书：即将支持，模块预留。
- 微博：即将支持，模块预留。
- TikTok：即将支持，模块预留。
- Instagram / Reels：即将支持，模块预留。
- 快手：即将支持，模块预留。

## 页面结构

整体采用三层布局：

- 背景层：保留当前 Canvas 粒子背景。
- 应用壳层：全屏深色仪表盘，左侧导航固定，右侧主内容滚动。
- 功能层：平台概览、解析输入、站内授权、结果区。

桌面端：

- 左侧宽度约 220px，包含品牌、平台导航、站点规则入口。
- 右侧主区域使用顶部平台概览卡片 + 下方控制台网格。
- 解析控制台作为主操作区，站内授权作为次级卡片。

移动端：

- 左侧导航变为顶部横向平台选择条。
- 控制台单列显示。
- 未支持平台的说明卡片保持可读，不占用过多首屏空间。

## 平台导航行为

每个平台配置包含：

- `id`：前端状态标识。
- `label`：展示名称。
- `status`：`supported` 或 `coming-soon`。
- `platform`：已支持平台对应后端枚举；未支持平台为空。
- `placeholder`：该平台链接示例。
- `cookieHint`：是否常需要平台 Cookie。
- `features`：能力标签，如“高清轨道”“浏览器合并”“需要 Cookie”“模块预留”。

点击左侧平台时：

- 高亮当前平台。
- 更新右侧标题、说明、链接 placeholder 和能力卡片。
- 如果平台已支持，解析按钮正常可用。
- 如果平台未支持，解析按钮禁用，显示“该平台即将支持”。

“全部平台”模式：

- 允许用户粘贴任意当前已支持链接。
- 显示三个已支持平台和多个预留平台的总览。

## 解析行为

后端解析流程不变：

- 已支持平台：继续调用 `/api/resolve`。
- 未支持平台：前端拦截，不调用后端，显示中文提示。
- 如果用户在某个已支持平台页面粘贴了其他平台链接，仍交给后端自动识别；前端不强制阻止。

## 视觉设计

参考用户提供的 AgentHub 风格，但适配视频解析产品：

- 保持黑色高级背景、玻璃卡片、荧光强调色。
- 左侧导航使用亮色激活态和小圆点状态。
- 不使用营销式大 Hero，首页第一屏就是可操作工作台。
- 平台卡片使用紧凑信息密度，适合频繁切换和扫描。
- 颜色不只依赖单一蓝紫，继续使用青绿、琥珀、玫红作为状态点缀。

## 组件计划

新增：

- `PlatformWorkspace`：客户端组件，管理当前平台选择、解析表单和结果状态。
- `platformCatalog.ts`：纯配置文件，定义平台列表、状态、文案和能力标签。

调整：

- `page.tsx`：保留 `ParticleField`，渲染 `PlatformWorkspace`。
- `LoginPanel`：放入工作台次级卡片。
- `ResolverForm`：可拆分或内联到 `PlatformWorkspace`，以便根据平台状态禁用提交。
- `ResultView` 和 `DownloadPanel`：继续复用。
- `globals.css`：新增左侧导航、平台卡片、工作台网格、移动端横向平台栏。

## 错误和提示

- 未支持平台：显示“该平台模块已预留，当前还未接入真实解析。”
- 需要账号态：继续提示粘贴平台 Cookie。
- 后端错误：继续显示 `/api/resolve` 返回的中文错误。
- 平台 Cookie 文案必须继续强调：访问码不是平台登录，Cookie 只用于本次解析。

## 验证

实现完成后需要验证：

- `npm test`
- `npm run typecheck`
- `npm run build`
- 公网页面包含小红书、微博、抖音等左侧模块。
- 已支持平台可正常提交解析。
- 未支持平台不会调用后端解析，并显示“即将支持”提示。
- 访问码和临时 Cookie 功能不回退。

## 后续扩展

未来可以按平台逐个接入真实 resolver：

- 抖音 resolver。
- 小红书 resolver。
- 微博 resolver。
- 快手 resolver。
- TikTok / Instagram resolver。

每个平台接入应单独写规格、测试和部署，避免把 UI 改造和解析能力混在同一次风险里。
