# 超级视频解析粒子控制台重设计规格

## 目标

把现有中文解析页面升级为“深色星图控制台”风格的高级工具页面。页面需要有明显但不过载的粒子交互：第一眼足够炫酷，同时保持解析链接、登录、复制命令、浏览器合并等核心流程清晰可用。

## 已确认方向

- 视觉方向：深色星图控制台。
- 动效强度：明显炫酷。
- 技术路线：Canvas 2D 粒子场，不引入 Three.js。
- 核心原则：服务器不代下载的产品定位不变，所有解析和下载功能不改接口、不改数据结构。

## 页面结构

首页保持单屏优先的工具站体验，不做营销落地页。

- 背景层：全屏 Canvas 粒子网络，位于页面最底层。
- 光效层：扫描光、细网格、边缘辉光，用 CSS 实现，不能遮挡文字。
- 主内容层：居中的控制台布局，包含标题区、状态条、登录区、解析表单和结果区。
- 结果区：沿用现有结果组件的功能，视觉改为深色卡片，按钮和命令块保持高对比度。

## 粒子交互

粒子系统使用独立客户端组件实现。

- 默认粒子缓慢漂移，形成星图网络。
- 鼠标移动时，指针附近粒子向鼠标轻微吸附，连线亮度增强。
- 鼠标离开窗口时恢复自然漂移。
- 粒子数量根据屏幕宽度和设备像素比控制，移动端自动减少。
- 页面不可见时暂停动画，回到页面后恢复。
- 用户系统开启 `prefers-reduced-motion: reduce` 时关闭强动效，只保留静态背景。

## 视觉设计

整体使用深色、多色点缀的科技工具风格，避免一整页只有单一蓝紫色。

- 背景主色：接近黑色的蓝黑和墨绿色。
- 强调色：青绿色、琥珀色、少量玫红色，用于粒子、按钮、状态点。
- 控制台：玻璃质感面板，8px 到 12px 圆角，边框细亮，阴影克制。
- 标题：中文大标题“超级视频解析”，辅助文案强调“服务器不代下载，浏览器本机下载/合并”。
- 状态条：显示“支持 YouTube / Bilibili / X”、“服务器不代下载”、“浏览器本机合并”等短标签。

## 组件调整

新增组件：

- `ParticleField`：客户端 Canvas 背景，负责粒子绘制和鼠标交互。
- `CapabilityStrip`：顶部能力状态条，展示关键产品规则。

调整组件：

- `LoginPanel`：改为深色控制台内的小型权限区，文案保留中文，访问码输入弱化为可信入口。
- `ResolverForm`：表单成为主控制台核心，输入框更醒目，按钮增加光泽和悬停反馈。
- `ResultView` 和 `DownloadPanel`：改为深色卡片风格，保持现有选择、直链、合并和复制命令流程。

## 数据流和行为

现有后端 API 和解析流程不变。

- 登录仍调用 `/api/login`。
- 解析仍调用 `/api/resolve`。
- 浏览器合并仍在用户本机执行。
- 服务端仍只返回解析清单，不代理视频文件。

## 错误处理

错误提示继续在表单下方显示中文信息。视觉上使用高对比但不刺眼的红色文本和边框。粒子背景不得影响错误文本可读性。

## 响应式

- 桌面端：标题区和主控制台形成宽屏布局，结果区使用网格卡片。
- 平板端：控制台收窄，状态条自动换行。
- 手机端：单列布局，粒子数量降低，输入框和按钮保持可点击尺寸。

## 可访问性和性能

- 所有输入、按钮和链接保留语义标签。
- 文本对比度优先于光效。
- Canvas 背景设置 `aria-hidden`，不进入辅助技术阅读流。
- 动画使用 `requestAnimationFrame`，组件卸载时清理事件和动画帧。
- 避免布局抖动，面板、按钮和结果卡片使用稳定尺寸约束。

## 测试和验证

实现完成后需要验证：

- `npm test`
- `npm run typecheck`
- `npm run build`
- 桌面和移动视口页面非空，粒子可见，核心表单可用。
- 公网部署后确认首页包含中文标题，服务状态正常。

## 非目标

- 不新增视频平台支持。
- 不改变访问码逻辑。
- 不引入服务端代下载、缓存、代理或合并。
- 不加入复杂营销页面、账号系统或支付功能。
