# 可复用工作流

目标：从参考素材分析到最终网站验收，形成可重复执行的高质量建站流程。这个流程适用于未来的高级网站设计任务，不绑定“超级视频解析网站”这个具体项目。

## 阶段 1：任务边界确认

输入：

- 用户目标
- 现有项目目录
- 参考图片、视频或网站
- 禁止修改范围

任务：

- 判断当前是审计、设计、实现还是验收。
- 明确哪些目录可以改，哪些业务逻辑不能动。
- 确认是否需要备份提交。

输出：

- 本次阶段目标
- 可修改范围
- 风险说明

验收条件：

- 不越权修改业务代码。
- 不把临时项目规则写成通用方法。

## 阶段 2：项目审计

输入：

- 项目源代码
- 现有截图
- README、测试和配置文件

任务：

- 检查技术栈、页面结构、组件结构、样式系统和数据流。
- 找出核心用户路径。
- 区分视觉问题、交互问题、代码结构问题。

输出：

- 当前信息架构
- 核心路径
- 值得保留的设计和工程部分
- 问题清单

验收条件：

- 结论有文件和截图依据。
- 不只说“高级感不足”，必须说明原因和改进方向。

## 阶段 3：参考素材处理

输入：

- 参考视频或图片
- 可用的视频处理工具

任务：

- 检查 `ffmpeg`、`ffprobe` 或其他视频处理工具。
- 对视频抽取代表性关键帧。
- 选出场景变化、设计变化、提示词展示、最终效果展示的帧。
- 不做逐帧 OCR。

输出：

- `keyframes/` 目录
- 关键帧索引
- 联系表或代表帧清单

验收条件：

- 关键帧覆盖完整流程。
- 文档引用具体帧，而不是根据文件名猜测。

## 阶段 4：提取设计 DNA

输入：

- 关键帧
- 参考图池
- 当前项目定位

任务：

- 分析色彩、字体、排版、留白、光影、层次、动效。
- 分离可复用方法和不可复制内容。
- 标注不确定信息。

输出：

- 设计 DNA 文档
- 可复用规则
- 项目专属规则

验收条件：

- 没有复制参考图的品牌、文案和完整布局。
- 能说明“为什么这样做”，不只列视觉形容词。

## 阶段 5：背景媒体生成规范

输入：

- 目标气质
- 可用参考图
- 页面内容结构

任务：

- 写背景图片提示词。
- 写背景视频/动效提示词。
- 明确哪些元素必须保持不变，哪些元素允许动态。
- 明确 AI 背景中禁止生成的内容。

输出：

- 背景图片提示词模板
- 背景视频提示词模板
- 素材验收标准

验收条件：

- 背景没有文字、按钮、导航、Logo 和假 UI。
- 背景有内容安全区。
- 真实 HTML 可以覆盖在背景之上。

推荐图片提示词结构：

```text
[产品/场景主题] background only,
[构图和镜头],
[光影和材质],
[色彩和氛围],
[为标题/表单预留的低细节区域],
high quality, responsive crop friendly,
no text, no letters, no buttons, no navigation, no logo, no UI
```

推荐视频提示词结构：

```text
Use the same background scene.
Ultra slow motion, subtle environmental movement, stable camera,
soft light shift, loopable, calm and unobtrusive.
No text, no UI, no logos, no fast camera movement.
```

## 阶段 6：真实 HTML 叠加设计

输入：

- 背景素材
- 页面真实信息
- 组件和样式系统

任务：

- 将标题、说明、按钮、输入框、导航和结果区作为真实 HTML 实现。
- 使用遮罩、渐变和安全区保证可读性。
- 复用已有组件和设计令牌。

输出：

- Section 设计说明
- 组件使用规则
- 分层规范

验收条件：

- 文字不烘焙进图片。
- 按钮和表单可以真实交互。
- 移动端不会因为背景裁切导致信息不可读。

## 阶段 7：逐个 Section 实现

输入：

- Section 设计说明
- 现有组件
- 背景素材

任务：

- 一次只实现一个 Section。
- 每个 Section 完成后检查桌面、移动端、动效和性能。
- 保持色彩、按钮、间距、边框、动效速度一致。

输出：

- 已实现 Section
- 对应截图或检查记录

验收条件：

- Section 自身完成。
- 与上下 Section 过渡自然。
- 没有卡片套卡片、文字重叠或布局跳动。

## 阶段 8：响应式和性能检查

输入：

- 已实现页面
- 背景图片/视频/Canvas

任务：

- 检查桌面、平板、移动端。
- 检查图片尺寸、视频体积、懒加载、降级方案。
- 检查 `prefers-reduced-motion`。
- 检查 Canvas 或 3D 场景是否非空、不卡顿。

输出：

- 响应式问题列表
- 性能风险列表
- 修复建议

验收条件：

- 文本不重叠、不溢出。
- 主操作在移动端可见。
- 背景媒体不阻塞核心功能。

## 阶段 9：最终验收

输入：

- 最终页面
- 原始目标
- 参考素材
- 项目规则

任务：

- 对照目标气质和用户路径验收。
- 检查是否复制了参考图具体内容。
- 检查是否改变了业务承诺。
- 运行必要测试和构建。

输出：

- 验收报告
- 剩余风险
- 后续任务清单

验收条件：

- 视觉升级不破坏功能。
- 通用方法和项目规则分离。
- 可清楚说明每个设计选择的原因。
