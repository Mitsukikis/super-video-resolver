## 视频展示的完整工作流

### 1. 先看最终质量目标

视频开头先展示高完成度页面，再倒推工作流。这个顺序很重要：目标不是“加粒子、加渐变、加卡片”，而是明确最终页面应该像一个完整产品：视觉背景、信息层级、导航、按钮和动效都围绕同一主题。

可复用方法：

- 先定义页面要给人的第一印象，例如精密、自然、速度、信任、创作力。
- 目标画面要包含真实信息层级，不只是一张好看的背景图。
- 成品验收必须看完整首屏，而不是只看单个组件。

### 2. 搜索和筛选参考图

视频在 Pinterest 类信息流中搜集大量参考，并横向浏览不同风格。能确认的做法是：先扩大搜索范围，再逐步收窄到与项目气质匹配的素材。

可复用方法：

- 不只搜“同类网站”，也搜海报、产品页、自然场景、摄影、3D、游戏 UI、字体样张。
- 参考图至少覆盖构图、色彩、字体、光影、动效、材质六个维度。
- 不以“这张好看”为唯一标准，而要问：它能提供哪种可迁移的设计 DNA。

待确认：

- 视频中每一次搜索的完整关键词无法完全辨认。
- 具体使用了哪些收藏夹或素材管理规则，视频没有完整展示。

### 3. 提取设计 DNA，而不是复制页面

视频反复强调不要照搬参考图的具体页面。参考图里出现的品牌、按钮、导航、文案、Logo 都不应该直接进入新项目。

可复用 DNA 包括：

- 构图：主视觉在画面中心、偏左或全屏铺底，前景和背景如何分层。
- 色彩：主背景的色相、明暗、饱和度、对比关系。
- 字体气质：衬线/无衬线、字重、字号对比、是否有编辑感。
- 光影：主光源位置、边缘光、体积光、发光强度。
- 留白：首屏信息的密度、按钮周围呼吸感、画面空区。
- 层次：背景媒体、雾化遮罩、真实 HTML 文案、操作按钮的顺序。

不能复制的内容：

- 参考图中的品牌、Logo、完整布局和文案。
- 参考图中的假导航、假按钮、假表单。
- 参考图中的可识别人物、受版权限制图片或完整商业素材，除非已授权。

### 4. 生成首屏背景图片或视频

视频中可见的提示词策略是：先从参考图抽出“背景”，并明确排除文字、按钮、导航和 Logo；再要求保留色调、构图和氛围，同时提高素材质量。

推荐提示词结构：

```text
主体/场景 + 构图 + 光影 + 色彩氛围 + 可放置 HTML 的空区 + 质量要求
+ 明确排除：no text, no letters, no buttons, no navbar, no logos, no UI
+ 输出约束：aspect ratio, resolution, background only
```

对动态背景的提示词结构：

```text
同一背景场景 + 极慢运动 + 轻微环境变化 + 可循环 + 镜头稳定
+ 明确排除：no text, no UI, no fast camera, no distracting objects
```

视频里能确认的动效方向包括：非常慢、柔和、自然、低幅度。它不是为了展示特效，而是让页面不死板。

### 5. 背景媒体与真实 HTML 分层

视频的核心方法之一是：背景媒体只承载氛围，文本、按钮、导航、表单全部由真实 HTML/CSS 实现。

原因：

- AI 生成的文字经常错字、变形、不可访问、不可响应。
- 假按钮和假导航不能交互，也会和真实 UI 重叠。
- 真正的网站需要可维护、可复制、可读屏、可响应式。

实现原则：

- 背景图/视频放在最低层，使用遮罩保证文字可读。
- HTML 内容使用项目真实文案、真实按钮、真实状态。
- 背景素材中预留暗区或低细节区，用于放标题、输入框或 CTA。
- 所有 UI 组件都应该能在无背景媒体时仍然成立。

### 6. 逐个 Section 实现

视频后段展示了逐段处理 Section、调整过渡、复用背景和组件的做法。它的意义是控制质量：每个 Section 都有自己的信息目标，同时仍然继承同一套视觉语言。

可复用方法：

- 每次只实现一个 Section，并验收它的层级、响应式和动效。
- 不要让 AI 一次生成整站完整页面；复杂页面要分段迭代。
- Section 之间应复用色彩、材质、按钮、间距、动效速度。
- 过渡区域要检查背景视频或图片的明暗是否突然跳变。

### 7. 验收与回看

视频中多次回到成品页面查看差距，说明高水平页面不是一次生成完成，而是循环调参。

建议验收项：

- 桌面端：首屏是否一眼说明产品，主操作是否明显。
- 移动端：标题、按钮、媒体和输入区域是否不重叠。
- 动效：是否过快、过亮、抢夺操作注意力。
- 性能：背景视频是否过大，是否提供静态封面或降级。
- 可维护：真实文案和按钮是否仍在代码里，而不是烘焙进图片。

## A. 可复用于其他网站的通用方法

- 先定义目标气质，再找参考，而不是看到什么好看就套什么。
- 建立参考池，横向比较后提取设计 DNA。
- 背景素材只生成环境、光影、材质和运动，不生成文字、按钮、导航。
- 所有业务内容都用真实 HTML/CSS 叠加。
- 逐个 Section 实现，每段都要有输入、输出和验收条件。
- 复用组件、色彩、动效速度和空间系统，保持整站一致。
- 用桌面端、移动端、性能、动效和可访问性共同验收。

## B. 仅适用于超级视频解析网站的项目规则

- 背景氛围应服务“解析控制台、可信工具、服务器不代下载”的定位，不应变成纯展示型艺术站。
- 主操作必须始终围绕“粘贴链接 -> 解析 -> 选择清晰度/下载方式”。
- 访问码和平台 Cookie 必须在视觉上分清：访问码只解锁本站功能，平台 Cookie 才是 X/YouTube/Bilibili 等账号态。
- 任何视觉升级都不能暗示本站会代理下载、保存视频、绕过 DRM 或绕过付费内容。
- 平台模块可以在 UI 中出现，但必须真实反映支持状态；未实现的平台应标记为待接入，而不是假装可用。
- 解析结果区必须优先展示直链下载、浏览器本地合并、本地工具命令三类动作的差异。
