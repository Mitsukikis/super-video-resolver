# 设计 DNA

这份文档把参考视频中的通用设计方法和当前项目的专属规则拆开。后续创建 Skill 时，通用方法可以进入 Skill；项目专属规则应留在项目文档或项目提示中，不应硬编码进通用 Skill。

## A. 通用建站方法

### 色彩系统

- 先确定品牌/产品气质，再确定主色，而不是先套深色渐变。
- 背景色、面板色、文字色、边框色、强调色、危险色、警告色必须成体系。
- 高级感通常来自克制的对比：大面积低饱和背景，小面积高饱和强调。
- 发光只能用于状态、焦点或关键操作，不应每个边框都发光。
- 同一页面不应被单一色相统治；需要至少一个功能性辅助色和一个警示色。
- 背景媒体必须经过遮罩处理，保证真实 HTML 文案的对比度。

### 字体系统

- 工具型产品优先使用清晰的系统无衬线字体。
- 中文界面建议准备专门字体栈，例如系统中文字体优先于英文 fallback。
- 展示型标题可以有更强个性，但表单、按钮、错误、结果必须高可读。
- 字号层级应少而稳：页面标题、区域标题、字段标签、正文、辅助说明、代码块。
- 不使用负字距；不靠超大字号掩盖层级问题。

### 间距系统

- 使用 4/8px 基准建立间距。
- 工具工作区应比营销页更紧凑，但不能拥挤。
- 同类组件的 padding、gap、label 距离要统一。
- 首屏应优先让主操作可见，而不是把空间全部给品牌表达。
- 固定格式组件要设置稳定尺寸，避免 hover、加载态或长文本导致布局跳动。

### 圆角和边框

- 卡片、面板和按钮圆角建议控制在 8px 以内，除非现有设计系统另有规定。
- 边框用于分层，不是装饰；同一层级的边框强度应一致。
- 重要面板可以有更强边框或阴影，但同屏不应太多元素同时“被强调”。
- 不要在卡片里继续嵌套装饰性卡片。

### 光影和背景

- 背景素材只负责氛围、空间和品牌情绪。
- 真实文字、按钮、导航、表单必须由 HTML/CSS 实现。
- AI 背景提示词必须排除文字、字母、按钮、导航、Logo、假 UI。
- 背景要预留内容安全区，避免复杂纹理压住标题或输入框。
- 动态背景应缓慢、低对比、可降级；不能影响阅读或操作。

### 布局和视觉层级

- 先画信息架构：导航、主操作、状态、结果、帮助分别在哪里。
- 工具型 Web 应用应优先工作区，而不是营销式 Hero。
- 左侧导航适合平台模块、功能模块和状态筛选。
- 顶部区域适合显示产品名、全局搜索、账号/授权、服务状态。
- 主区域应给核心任务最大空间。
- 结果区应在任务完成后成为视觉中心。

### 动效原则

- 动效要解释状态或增加空间质感，而不是单纯炫技。
- 背景粒子、视频、光扫线都要支持 `prefers-reduced-motion`。
- 鼠标互动可以增强高级感，但不能改变布局或遮挡输入。
- Section 过渡要检查色调、明暗和节奏，避免断层。
- 动效验收应包含桌面、移动端和低性能设备。

### 需要避免的设计问题

- 复制参考图的品牌、文案、完整布局。
- 使用带文字、按钮、导航、Logo 的 AI 背景作为真实页面。
- 过多卡片、过多发光、过多边框。
- 只做首屏，后续 Section 视觉断裂。
- 背景很华丽但主操作不明显。
- 一味追求暗色科技风，忽略业务可信度和可读性。
- 让视觉改造改变产品真实承诺。

## B. 超级视频解析网站项目规则

### 产品定位

这是一个视频解析工具，不是视频托管、视频代理或代下载服务。视觉设计必须让用户相信它轻量、可信、可控，而不是暗示平台会替用户下载大量视频。

### 平台模块

适合采用左侧模块导航，模块包括：

- 总览/全部平台
- X / Twitter
- Bilibili
- YouTube
- 抖音（待接入）
- 小红书（待接入）
- 微博（待接入）

未实现的平台必须明确显示“待接入”或“规划中”。不能在 UI 上制造可解析的假状态。

### 核心路径

主路径必须稳定：

```text
选择平台 -> 粘贴链接 -> 可选粘贴平台 Cookie -> 解析 -> 选择下载方式 -> 用户本机下载或合并
```

任何视觉升级都不能让这条路径变长或变模糊。

### 访问码和 Cookie

- 访问码：只解锁本站受控功能，例如临时 Cookie 输入。
- 平台 Cookie：来自用户自己的 X/YouTube/Bilibili 等账号，只用于本次解析。
- 页面必须避免让用户误以为输入访问码就等于登录视频平台。
- Cookie 输入区域应按平台给出示例和风险说明。

### 结果与下载

结果区必须清晰说明：

- 直链下载：浏览器直连源站，不经过本站服务器。
- 浏览器合并：音视频分离时在用户本机浏览器合并。
- 本地工具：复制 ffmpeg、yt-dlp、aria2 等命令在本机执行。
- 敏感命令：包含 Cookie 时必须有明显风险提示。

### 安全和合规

- 不提供 DRM、私密、付费墙、会员限定内容的绕过。
- 不持久化保存用户平台 Cookie。
- 不在服务器下载、缓存、合并或代理视频文件。
- 错误文案应帮助用户理解限制，而不是鼓励规避平台规则。

### 当前项目应保留的 DNA

- 深色技术工具气质。
- 服务器不代下载的安全承诺。
- 浏览器本地合并的核心卖点。
- 临时 Cookie 不保存的信任表达。
- 统一 Manifest 的结果结构。

### 当前项目需要增强的 DNA

- 平台工作区：让多平台能力成为第一层结构。
- 结果工作台：让解析成功后的操作更像专业控制台。
- 状态表达：平台可用性、Cookie 要求、失败原因、速率限制要前置。
- 排版质感：减少对发光和边框的依赖，用字体、留白和层级建立高级感。
- 动效克制：粒子和背景应衬托，而不是抢走输入区注意力。
